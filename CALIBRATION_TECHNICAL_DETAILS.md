# Mini45 三维力传感器标定技术说明

本文档用于记录本项目上位机在标定实验中使用的主要技术细节，便于后续整理毕业论文中的实验系统、控制算法和数据采集方法章节。README 只说明项目运行方式，本文档说明算法和实验实现。

## 1. 实验系统组成

标定系统由四部分组成：

- 待标定柔性电容式三维力传感器：输出五通道电容 `C0~C4`。
- ESP32 + MC1081 采集电路：通过 USB 串口向上位机发送电容数据。
- ATI Mini45 六轴力传感器：通过 NETBA 盒和以太网向上位机输出 `Fx,Fy,Fz,Mx,My,Mz`。
- Arduino Mega 三轴电机控制器：通过 USB 串口接收上位机 `MOVE_MM` 指令，控制三轴丝杆台移动。

上位机使用同一台 PC 的系统时钟为 Mini45 和 ESP32 数据打时间戳。静态标定数据和训练采集数据分文件保存，后续时间戳匹配、滤波、零点扣除和模型训练均在独立数据分析项目中完成。

## 2. 坐标与控制对象定义

Mini45 的原始坐标系不一定与待标定传感器坐标系一致。上位机在 Mini45 数据进入主数据流前，先根据界面中的“传感器坐标映射”将 Mini45 原始力转换为传感器坐标力：

```text
F_sensor = T F_mini45
```

当前实现的 `T` 是轴置换加符号翻转矩阵，即每个传感器力轴只能从一个 Mini45 力轴取得，并乘以 `+1` 或 `-1`。例如：

```text
传感器 Fx = - Mini45 Fz
传感器 Fy = + Mini45 Fy
传感器 Fz = + Mini45 Fx
```

转换后的 `Fx,Fy,Fz` 被作为闭环控制、K 辨识、稳定判定和标定输出使用的统一力坐标。Mini45 原始力/力矩仍会以 `mini45_raw_*` 字段保留在原始时序文件中，便于追溯。

Arduino 电机轴固定记为 `X,Y,Z`。在当前标定台结构中，电机物理轴与力主响应方向大致为：

```text
Arduino X 轴主要影响 Mini45 Fz
Arduino Y 轴主要影响 Mini45 Fy
Arduino Z 轴主要影响 Mini45 Fx
```

该关系只用于人工理解和手动小步调试。正式自动力控不再使用“某个力轴误差只移动某个电机轴”的单轴映射，而是通过自动辨识得到的完整 `3×3` 力-位移灵敏度矩阵统一计算三轴位移。

## 3. 力-位移灵敏度矩阵 K

上位机在正式自动力控前自动辨识矩阵 `K`。`K` 的单位为 `N/mm`，行固定为传感器坐标三向力，列固定为 Arduino 三个电机轴：

```text
K =
[
  [K_Fx_X, K_Fx_Y, K_Fx_Z],
  [K_Fy_X, K_Fy_Y, K_Fy_Z],
  [K_Fz_X, K_Fz_Y, K_Fz_Z]
]
```

其中 `K_Fx_X` 表示 Arduino `X` 轴正向移动 1 mm 引起传感器坐标 `Fx` 的变化量。每一列都包含传感器坐标 `Fx,Fy,Fz` 三个分量，因此矩阵可以记录三轴加载之间的耦合关系。

对任一电机轴 `J`，辨识流程为：

```text
1. 采集扰动前传感器坐标三向力均值 F_before,J。
2. 发送 MOVE_MM J δJ speed，使该轴正向移动小位移 δJ。
3. 等待运动完成和短暂稳定。
4. 采集扰动后传感器坐标三向力均值 F_after,J。
5. 计算 K 的第 J 列：
   K[:,J] = (F_after,J - F_before,J) / δJ
6. 发送 MOVE_MM J -δJ speed，使该轴回退到扰动前位置附近。
```

默认扰动位移为 `δX=δY=δZ=0.10 mm`，扰动后等待时间为 `1.0 s`，均值采样窗口为 `1.0 s`。如果接触风险较高，可将扰动位移减小到 `0.02~0.05 mm`；如果力变化接近噪声或 K 辨识结果导致自动力控长期微小步进，可适当增大扰动位移以提高 K 的辨识信噪比。

## 4. K 的有效性判断

自动辨识完成后，上位机会检查 `K` 是否适合用于正式自动力控。

有效性判据包括：

- 每个电机轴扰动引起的三向力变化范数必须足够大：

```text
||ΔF_J|| >= max(0.05 N, 3 × 当前采样窗口噪声)
```

- `K` 必须满秩，避免某个电机轴对力没有独立控制作用。
- `K` 条件数默认要求不超过 `300`。条件数过大说明矩阵病态，控制量会对噪声和微小误差非常敏感。
- 辨识过程中如果 `|Fx|>4 N`、`|Fy|>4 N`、`|Fz|>10 N` 或力矩超过阈值，上位机会立即停止电机并终止辨识。

有效的 `K` 会缓存在当前上位机运行状态中，并写入批次文件夹下的 `force_control_k.csv`。重新安装、重新粘接或接触状态明显变化后，应重新辨识 `K`。

在上位机界面中，`力控参数` 属于实验批次级设置，不属于某个单独实验模式。完成一次有效 K 辨识后，同一批次内的单目标点标定、静态正反程标定和训练数据采集都会复用该 K。

## 5. 解耦位移增量闭环控制

设当前传感器坐标三向力为：

```text
F(k) = [Fx(k), Fy(k), Fz(k)]^T
```

目标力为：

```text
Ft = [Fx_t, Fy_t, Fz_t]^T
```

力误差为：

```text
e(k) = Ft - F(k)
```

在当前接触状态附近，三轴电机位移增量与传感器坐标三向力变化近似满足：

```text
ΔF(k) = K Δs(k)
```

其中：

```text
Δs(k) = [ΔX(k), ΔY(k), ΔZ(k)]^T
```

上位机根据 `K` 和误差 `e(k)` 计算理论电机位移增量。输出的 `ΔX,ΔY,ΔZ` 直接对应 Arduino `X,Y,Z` 电机轴，不再经过旧的力轴到电机轴单轴映射。

## 6. 自动阻尼与信任域控制

由于柔性传感器接触系统存在非线性、迟滞和噪声，直接求解 `K Δs=e` 容易过冲。上位机采用阻尼最小二乘思想计算位移增量：

```text
Δs_raw = (K^T K + η^2 I)^(-1) K^T e
```

其中 `η` 为自动阻尼项。程序根据 `K` 的奇异值、条件数和辨识噪声自动确定阻尼强度：

- `K` 条件数越大，阻尼越强。
- 辨识噪声越大，阻尼越强。
- 矩阵状态较好时，阻尼保持较小，以提高控制响应。

理论位移 `Δs_raw` 不会直接发送给电机，而是经过信任域限制：

- 单轴位移不超过 UI 中设置的最大单步位移。
- 预测力变化不超过当前误差和安全余量。
- 误差下降明显时，信任域适度放大。
- 误差长期下降不足时，信任域会逐步放大，避免机械间隙或偏保守 K 导致长时间小步爬行。
- 误差增大或出现换向时，信任域自动减小。
- 当目标力误差仍较大时，可启用最小有效步长限制，使输出位移不长期停留在 `0.005 mm` 这类机械上不明显的量级；接近目标窗口后该限制不再强制放大位移。
- 接近目标窗口时，实际位移自然减小。

UI 提供 `保守 / 标准 / 快速` 三种控制风格，默认使用 `标准`。如果加载过程仍过慢，可增大 `最大单步 mm`、`最小有效步 mm` 或切换到 `快速`；如果出现过冲或振荡，应降低这些参数或切换到 `保守`。

## 7. 脉冲量化与电机指令

当前丝杆台参数为：

```text
丝杆导程 = 2 mm/rev
驱动脉冲 = 400 pulse/rev
位移分辨率 = 0.005 mm/pulse
```

上位机计算得到连续位移后，先做单步限幅，再按 `0.005 mm/pulse` 量化为脉冲。若三轴量化后均为 0，但力误差仍未进入容差窗口，程序会对理论位移最大的轴补偿 1 个脉冲，避免闭环停滞。

最终发送给 Arduino 的指令仍为现有协议：

```text
MOVE_MM X delta_x speed
MOVE_MM Y delta_y speed
MOVE_MM Z delta_z speed
```

因此本方案不需要修改 Arduino 串口协议，也不需要修改 `arduino_motion_serial.ino`。

## 8. 静态标定流程

静态标定用于计算论文中的灵敏度、线性度、迟滞、重复性、回零误差等指标。推荐流程为：

```text
1. 开始实验批次。
2. 执行空载零点漂移。
3. 自动辨识 K。
4. 执行 Fz 静态正反程标定。
5. 执行 Fx 静态正反程标定。
6. 执行 Fy 静态正反程标定。
7. 结束实验批次。
```

每个目标点首先使用严格三向力窗口 `Fx/Fy ±0.05 N、Fz ±0.08 N` 判断到达，并连续稳定保持 5 s。随后采集 45 个完整电容帧。为避免低频电容采集接近完成时因瞬时力波动丢弃全部数据，采集阶段使用具有迟滞特性的保持窗口：

```text
采集保持窗口 = max(严格进入窗口 × 1.5, 对应力轴满量程 × 1.5%)
```

按 `Fx/Fy` 满量程 4 N、`Fz` 满量程 10 N 计算，默认采集保持窗口约为 `Fx/Fy ±0.075 N、Fz ±0.150 N`。采集期间短暂超出保持窗口时暂停接收新的电容样本，并保留已采数据；若 5 s 内恢复则继续采集。持续越界超过 5 s 且采集进度低于 80% 时清空并重新稳定；进度达到 80%（默认至少 36/45 帧）时继续保留已有样本并等待力恢复，直到完成或触发单点总超时。最终标定点仍需通过严格均值、力标准差和电容质量判定。

上位机将算术均值、标准差以及逐字段剔除 P01/P99 外极端值后的 trimmed mean 写入 `calibration_points.csv`。论文中的静态指标优先使用 `*_trimmed_mean`，原始均值和标准差用于质量分析。

## 9. 训练数据采集流程

训练数据采集用于后续深度学习实时力预测模型。该模式以转换后的待测传感器坐标力作为目标和到达判断依据，不使用 Mini45 原始坐标力作为训练标签。

训练模式采用“目标队列 + 实测到达切换”：

- 上位机生成 `Fx往返`、`Fy往返`、`斜向往返` 或 `随机扰动` 的目标队列。
- 每个目标开始时写入 `target_start`，然后使用 K 解耦力控连续逼近该目标。
- 当转换后的 `Fx,Fy,Fz` 都进入训练到达窗口时，立即写入 `target_reached` 并切换下一目标。
- 训练目标不要求稳定保持，也不生成静态标定点。
- 如果目标超过最大等待时间仍未到达，默认 60 s 后写入 `target_timeout` 和 `target_skipped`，再进入下一目标。
- 随机扰动每次运行生成新的圆盘内部随机剪切目标，用于补充实际力空间覆盖。

训练过程中保存原始异步时序：

- 转换后的传感器坐标力/力矩和 Mini45 原始力/力矩写入 `training_raw_timeseries.csv`。
- ESP32 电容原始数据写入同一文件。
- 轨迹阶段写入 `training_markers.csv`。

该模式不在上位机中做时间戳匹配、插值、滤波、零点扣除或标准化。所有训练预处理应放在后续深度学习项目中完成。

## 10. 输出文件用途

```text
raw_timeseries.csv
  静态标定过程中的传感器坐标力、Mini45 原始力和 ESP32 原始异步时序。

markers.csv
  静态标定事件 marker。

calibration_points.csv
  静态稳定点统计结果，用于论文静态性能指标。

zero_drift_timeseries_XXX.csv
  单次空载零点漂移原始时序。

training_raw_timeseries.csv
  深度学习训练采集模式的原始异步时序。

training_markers.csv
  训练轨迹阶段 marker。

force_frame_mapping.csv
  本批次 Mini45 到传感器坐标的轴向和符号映射。

force_control_k.csv
  传感器坐标下的 K 矩阵辨识参数、结果和有效性判断。

force_control_log.csv
  自动力控每一步的误差、位移、脉冲和控制状态。
```

## 11. 论文写作建议

论文中可以将本上位机方法概括为：

```text
本研究采用 ATI Mini45 作为标准力传感器，通过三轴电动位移平台对柔性电容式三维力传感器施加载荷。上位机首先根据安装关系将 Mini45 原始坐标转换为待标定传感器坐标，并在正式标定前自动辨识电机位移与传感器坐标三向力之间的局部灵敏度矩阵。随后基于阻尼最小二乘和自适应信任域策略计算三轴电机位移增量，实现目标力的自动逼近。实验数据采用 Mini45 与电容采集的连续时间戳记录方式保存，静态指标由稳定窗口统计点计算，深度学习训练数据则保留完整连续原始时序。
```
